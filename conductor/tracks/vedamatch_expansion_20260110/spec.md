# Спецификация: VedaMatch & Networking (Расширенные знакомства)

## 1. Обзор
Трансформация VedaMatch из сервиса только для брачных знакомств в универсальную платформу для нетворкинга в благости. Пользователи могут переключать "режимы" поиска или указывать несколько целей в своем профиле: Семья, Бизнес/Работа, Дружба/Интересы, Служение.

## 2. Цели
- **Многоцелевые профили**: Позволить пользователям заполнять разные блоки информации для разных целей (например, "Духовная практика" для семьи, "Навыки" для бизнеса).
- **Режимы поиска**: Фильтры и алгоритмы, адаптированные под цель (Бизнес = поиск по навыкам/отрасли; Семья = поиск по ашраму/гороскопу).
- **Мэтчинг по интересам**: Поиск людей для совместных проектов или служения.

## 3. Пользовательские истории
- **Семья**: "Я ищу жену, которая разделяет мои стандарты чистоты и практику."
- **Бизнес**: "У меня вегетарианское кафе, ищу управляющего-вайшнава." или "Ищу партнера для IT-стартапа в благости."
- **Служение**: "Ищем пуджари или повара на фестиваль."
- **Интересы**: "Ищу, с кем вместе воспевать киртан или заниматься хатха-йогой в моем районе."

## 4. Определение API (Расширение)
- `POST /api/dating/profile`: Обновление профиля с новыми секциями (business_info, seva_interests).
- `GET /api/dating/search`: Параметр `mode` (family, business, friends, seva).
- `POST /api/dating/like`: Лайк/Коннект с указанием цели ("Интересно сотрудничество").

## 5. Технические детали
- **Модель данных**: Расширение таблицы `users` или `dating_profiles` полями JSONB или отдельными таблицами:
    - `Intentions` (enum: marriage, business, friendship, seva).
    - `BusinessProfile` (skills, industry, looking_for).
    - `Interests` (tags).
- **Алгоритм**:
    - Если mode=business, скрывать гороскоп, показывать навыки.
    - Если mode=family, показывать духовную практику и астрологию.
