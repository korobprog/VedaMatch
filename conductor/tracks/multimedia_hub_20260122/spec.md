# Спецификация: Мультимедиа-хаб (Sattva Media)

## Обзор
Создание единого мультимедийного пространства внутри приложения, объединяющего аудио (бхаджаны, лекции), видео (кинотеатр), онлайн-радио и круглосуточное духовное ТВ. Система должна обеспечивать бесшовный пользовательский опыт с поддержкой фонового воспроизведения и оффлайн-доступа.

## Функциональные требования

### 1. Точки входа и навигация
- Иконки сервисов на главном экране для быстрого доступа к разделам: Радио, Аудио, Кинотеатр, ТВ.
- Глобальный мини-плеер в нижней части экрана для управления воспроизведением во время навигации по приложению.

### 2. Категории и контент
- **Радио**: Потоковое вещание духовных радиостанций.
- **Аудио (Бхаджаны/Лекции)**: Коллекция записей с фильтрацией по матхам (организациям), стилям исполнения и авторам.
- **Кинотеатр (Видео)**: Видео-библиотека с фильмами и записями программ.
- **ТВ**: Прямые трансляции и круглосуточные каналы.

### 3. Возможности плеера
- **Аудио и Видео**:
    - Фоновое воспроизведение (для аудио и радио).
    - Выбор скорости воспроизведения (0.5x, 1x, 1.5x, 2x).
    - Таймер сна.
    - Оффлайн-режим (скачивание аудио/видео в локальное хранилище).
    - Управление качеством видео потока.
    - Полноэкранный режим для видео.
- **Социальные функции**:
    - Создание и управление пользовательскими плейлистами.
    - Функция «Поделиться» (внутренняя ссылка пользователю приложения и внешняя ссылка в соцсети/мессенджеры).

### 4. Управление контентом
- **Админ-панель**: Загрузка файлов (S3) и добавление ссылок на потоки администраторами.
- **UGC (Пользовательский контент)**: Интерфейс для предложения контента пользователями с обязательной премодерацией перед публикацией.

## Технические требования
- **Хранилище**: Использование S3-совместимого хранилища для всех медиафайлов (видео и аудио).
- **Стек**: 
    - Frontend: React Native (использование библиотек типа `react-native-video` или `react-native-track-player`).
    - Backend: Go (Fiber) для метаданных и управления доступом к S3.

## Приемочные критерии
1. Пользователь может запустить аудио и свернуть приложение, при этом звук продолжает играть.
2. Видео корректно воспроизводится из собственного хранилища S3.
3. Доступны фильтры по матхам в разделах аудио и видео.
4. Файлы, скачанные для оффлайн-режима, доступны без подключения к интернету.
5. Контент, предложенный пользователем, появляется в общем доступе только после одобрения в админ-панели.

## Вне рамок (Out of Scope)
- Прямой стриминг (Live Broadcast) с мобильного приложения пользователя (только просмотр готовых трансляций).
- Платные подписки или аренда контента (на данном этапе весь контент бесплатный).
